## 桌面端 UI 与设备可见性联调

- 需求：主界面需要直接显示网关已注册设备，不能仅依赖日志判断。
- 决策：复用网关 `/v2/clients` 数据源，桌面端每 2 秒轮询一次并渲染到“已注册设备”面板。
- 实现：
  - 新增 `GatewayClientsSnapshotParser`，统一解析 `ClientListResponse.items` 与回调端口抽取逻辑。
  - `WindowController` 的 `ViewModel` 增加 `registeredClients`/`clientsSummaryText` 状态，展示平台、appId、sessionId、deviceId、lastSeen、selected。
  - `HarmonyPortBridgeManager` 回调端口解析改为复用同一解析器，避免 JSON 结构重复维护。
- UI 结果：在运行状态下可见设备列表；无设备时显示空态提示；`Client` 状态卡由轮询结果更新（已注册设备数/选择数）。
- 验证：`swift build` 通过，桌面端可启动并持续收到 `/v2/clients` 请求日志。
- 风险：当前 `swift test` 受仓库内已有 `USBMuxd*` 相关缺失符号影响，无法作为本次改动的完整门禁，需后续单独修复测试基线。

## 鸿蒙模拟器网关发现失败修复

- 现象：鸿蒙模拟器偶发发现网关失败，桌面端端口冲突时网关会自动从 `18765` 漂移到其它端口（例如 `18767`），而模拟器侧常用回退 DSN 仍是 `127.0.0.1:18765`。
- 决策：在 Harmony 端口桥接层增加“设备端口别名”能力，保持 `18765` 永远可达（转发到实际网关端口）。
- 实现：
  - `HarmonyPortBridgeManager.Configuration` 新增 `gatewayAliasPorts`。
  - 桥接命令从固定 `tcp:X -> tcp:X` 改为支持 `tcp:devicePort -> tcp:hostPort`。
  - `main.swift` 在网关端口发生漂移时自动追加 `18765` 别名桥接。
- 验证：
  - `swift build` 通过。
  - 在占用 `18765` 的情况下启动桌面端，网关自动漂移到 `18766`。
  - `hdc fport ls` 可看到 `18765` 和 `18766` 的 reverse 规则同时存在，说明别名桥接已挂上。
- 预期结果：即便桌面端网关端口自动切换，鸿蒙模拟器使用 `127.0.0.1:18765` 仍可完成 discovery/register。
