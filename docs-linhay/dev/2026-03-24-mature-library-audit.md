# 成熟库优先审计记录

- 仓库：`neptune-desktop-macos`
- 日期：2026-03-24
- 结论：当前实现没有发现需要强制替换的 SQLite / HTTP / CLI / 协议解析自研基础设施。

## 审计范围

重点检查以下类型的手搓基础能力：

- 进程管理
- 参数解析
- 网络通信
- 协议解析
- 数据持久化

## 结果

### 1. 无 SQLite / HTTP / CLI / 协议解析实现

仓库当前只有一个极薄的 macOS shell：

- `main.swift` 负责 AppKit 生命周期
- `WindowController.swift` 负责 `WKWebView` 容器
- `GatewayLauncher.swift` 负责启动本地网关进程

没有发现：

- SQLite 自研封装
- HTTP 客户端 / 服务端自研实现
- CLI 参数解析器
- 自定义协议 parser / serializer

### 2. 进程启动逻辑是唯一的手写基础设施

`GatewayLauncher.swift` 使用系统 `Process` / `Pipe` / `NSLock` 直接管理网关进程，并通过环境变量读取启动参数。

判断：

- 这是必要的薄封装，不属于可替换成高收益三方库的场景
- 当前代码量小，职责单一，没有明显的抽象膨胀
- 暂不引入额外依赖，避免把一个简单 shell app 复杂化

### 3. 现状风险

- 进程输出只做了按行日志转发，缺少结构化诊断
- 网关启动失败时，主窗口仍会打开，属于“尽力启动”的 UX
- 若后续增加 CLI 参数、子进程编排或多进程生命周期管理，再评估成熟库或更强的进程封装

## 结论

当前仓库符合“成熟库优先”的基本要求，因为没有把 SQLite、HTTP、CLI 或协议解析这类基础能力手搓在仓库里。

本次不新增依赖，不改核心逻辑，仅保留现有薄壳实现。
